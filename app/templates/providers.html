How Providers Work

Chat provider — generates the actual responses the auto-pilot sends. Needs a good conversational model (GPT-4o-mini, Claude Haiku, etc.)

Supervisor provider — reviews messages before sending, analyzes bans, learns patterns. Can use a cheaper/faster model since it's just classification.

Fallback provider — used when the main provider fails. Keep a different provider type here (e.g., Anthropic as fallback for OpenAI).

💡 Recommended: OpenRouter with one chat provider + one supervisor. Use different models for each role.
